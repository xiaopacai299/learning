## L1 · 基础自检（简单 · 快过）

**目标**：确认没有「会用但说不清」的硬伤，面试不会在第一轮翻车。

### 1.1 核心概念

- 组件、JSX（表达式、`{}`、为何避免随意 `{}` 新对象）、Props 只读、单向数据流。  
- State 与 setState / `setX`：**异步批量更新**（React 18 自动批处理）。  
- 类组件 vs 函数组件（面试知道差异即可，**以函数组件 + Hooks 为主**）。

### 1.2 面试题清单

### 1.3 易错点

- 把「虚拟 DOM」说成「一定比直接操作 DOM 快」。

## L2 · 渲染与更新模型（简单+ · **必扎实**）

**目标**：能解释「何时渲染、渲染什么、diff 在干什么」——大厂一面核心。

### 2.1 核心概念

- **Render**：根据 state/props 计算 UI 描述（函数组件即函数执行）。  
- **Reconciliation**：对比新旧 Element 树，得出变更。  
- **Commit**：把变更应用到 DOM（及 layout、passive effects）。  
- 默认策略：**父 render → 子默认跟着 render**（除非被 memo / 优化阻断）。  
- Diff 假设：**同层比较**、**类型不同则整棵替换**、**列表用 key 标识稳定身份**。

### 2.2 面试题清单

| 题 | 难度 |
|----|------|
| 组件什么情况下会重新渲染？ | ⭐ |
| 为什么父组件更新会导致子组件渲染？如何减少？ | ⭐⭐ |
| 虚拟 DOM + Diff 的流程（用自己的话 30 秒版） | ⭐⭐ |
| `key` 在 Diff 中的作用；为什么不推荐随机 key | ⭐⭐ |
| `setState` 是同步还是异步？React 18 批处理 | ⭐⭐ |
| Element、Component、Fiber 三者区别（先建立词汇） | ⭐⭐⭐ |

### 2.3 易错点

- 认为「props 没变子组件就一定不渲染」（未考虑 **Context、父 render**）。  
- 把 **Render 阶段** 和 **Commit 阶段** 的副作用混为一谈。

### 2.4 与 `a.md` 对照

- 渲染机制、父组件更新：`a.md` §5（2）渲染机制。

## L3 · Hooks 体系（中等 · **面试主战场**）

**目标**：不仅会写，还能讲清依赖、闭包、清理函数与规则。

### 3.1 核心概念

- **Hooks 规则**：仅顶层调用；仅 React 函数组件 / 自定义 Hook 内调用。  
- `useState` / `useReducer`：状态与派发。  
- `useEffect`：副作用；依赖数组三种形态；清理函数与 **严格模式双调用**（开发环境）。  
- `useLayoutEffect` vs `useEffect`：与浏览器绘制时机。  
- `useRef`：DOM 引用、**跨渲染可变值且不触发渲染**。  
- `useMemo` / `useCallback`：**缓存的是引用/计算结果**，不是「少渲染」的银弹。  
- 自定义 Hook：抽逻辑，名字 `use` 开头。

### 3.2 面试题清单

| 题 | 难度 |
|----|------|
| Hooks 相比 class 的生命周期解决了什么？ | ⭐ |
| `useEffect` 依赖数组为空、省略、写全的区别 | ⭐⭐ |
| 闭包陷阱：effect 里拿到旧 state 的原因与修法 | ⭐⭐ |
| `useMemo` / `useCallback` 一定提升性能吗？ | ⭐⭐ |
| `useRef` 和 `useState` 区别 | ⭐ |
| 自定义 Hook 设计原则（举例：useToggle、useFetch） | ⭐⭐ |
| `useEffect` vs `useLayoutEffect` | ⭐⭐ |
| 为什么不能在条件里写 Hook？ | ⭐⭐ |

### 3.3 易错点

- 依赖数组漏项导致 **stale closure**。  
- 到处 `useCallback` 反而增加比较成本。  
- 把 `useMemo` 当成「所有昂贵计算都要包」。

### 3.4 与 `a.md` 对照

- Hooks 表、useEffect 对照：`a.md` §3 hooks。

### 5.1 核心概念

- 单向数据流、状态提升、**组合优于继承**。  
- Context 适用：**中低频、跨多层** 的全局配置（主题、语言、当前用户）。  
- HOC、Render Props、自定义 Hook：**逻辑复用** 三代演进（面试常问对比）。  
- 受控表单、非受控 + ref；复杂表单可提 **react-hook-form**（了解即可）。

## L8 · 大厂原理与工程（难 · **二面 / 资深向**）

**目标**：能画 **Fiber 调度主流程**、说清 **双缓冲**、**Commit 三子阶段**；结合项目讲 SSR/水合。

### 8.1 核心概念（按优先级）

1. **Fiber**：可中断的工作单元；链表结构；`current` / `workInProgress` 双树。  
2. **调度**：lane 优先级（知道概念）；`requestIdleCallback` 到 `MessageChannel` 的演进可简述。  
3. **Render phase vs Commit phase**：Render 可中断，Commit 同步刷 DOM。  
4. **Hydration**：SSR HTML + 客户端对齐，不匹配问题。  
5. **状态管理选型**：本地 state / Context / Zustand / Redux（面试讲 **适用场景与缺点**）。  
6. **Next.js**：SSR、SSG、ISR、RSC 名词（按岗位）。
