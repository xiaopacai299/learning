## 2026-05-19 · 只改用户昵称，PUT 还是 PATCH？

**你的答案**：选 A（PUT），因为 PUT 幂等；不清楚两者差别和 REST 含义。

**正解**：**PATCH /users/1** 只改部分字段更合适。**PUT** 语义是 **整体替换** 该资源，应带完整表示；只传 `name` 可能把其它字段置空（看服务端实现），易踩坑。两者都可设计成幂等，但 **部分更新用 PATCH**。

**解析**：幂等不是选 PUT 的唯一理由；PATCH 也可幂等。REST 是用 URL 表资源 + HTTP 方法表动作，无状态交互。

**要点**：PUT = 全量替换；PATCH = 局部更新；GET 查 POST 建 DELETE 删。

## 2026-05-19 · Express 中间件 `next()` 洋葱打印顺序？

**你的答案**：5 1 234。

**正解**：**1 → 3 → 5 → 4 → 2**（`13542`）。

**解析**：请求从上往下进：先 1，`next()` 进 3，再 `next()` 到路由 5；`res.send` 后开始「出洋葱」：3 里 `next()` 后面的 4，再 1 后面的 2。5 不会最先执行。

**要点**：进栈顺序 1-3-5；出栈顺序 4-2；`next()` 后是「回来」才执行的代码。

## 2026-05-19 · 主文件顶层 setTimeout(0) 与 setImmediate 谁先？

**你的答案**：场景 1 一定 B（setImmediate）先。

**正解**：场景 1 **不一定**，A、B 顺序 **可能互换**（和进程启动、定时器是否已注册有关）。场景 2 **D（setImmediate）先** 是对的。

**解析**：主文件顶层跑完同步代码后，Event Loop 第一次转：timer 与 check 阶段谁先后不固定。I/O 回调里本轮 poll 刚结束，**先 check（setImmediate）再进 timers**，故 D 稳先于 C。

**要点**：顶层 = 顺序不确定；I/O 回调内 = setImmediate 先于 setTimeout(0)。
